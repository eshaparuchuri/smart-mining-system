# Smart Mining Digital Twin System

## Overview
The Smart Mining Digital Twin System is a comprehensive end-to-end intelligent platform designed to simulate, monitor, and analyze mining machinery in real time through the concept of a digital twin. A digital twin is essentially a virtual representation of a physical system, and in this project, it is used to replicate the behavior and condition of mining equipment using continuously generated data and predictive models. The primary objective of this system is to enable proactive monitoring and predictive maintenance by combining real-time data simulation, machine learning, and interactive visualization into a unified pipeline.
At the core of the system lies a data simulation layer that mimics real-world industrial sensor outputs. Since access to actual mining equipment data is often limited, a custom-built simulator generates realistic time-series data representing critical machine parameters such as temperature, vibration, load, and pressure. These parameters are carefully designed to behave like real operational signals, including normal fluctuations and occasional abnormal spikes to simulate fault conditions. This simulated data acts as the foundation upon which the rest of the system operates, effectively replacing live IoT streams in a controlled environment.
The generated data is then fed into a backend system implemented using Flask, which acts as the central processing unit of the digital twin. The backend exposes APIs that receive incoming sensor data, process it through trained machine learning models, and return structured outputs that describe the machine’s condition. One of the key components here is the anomaly detection model, which evaluates whether the incoming data represents normal behavior or indicates a potential issue. Alongside classification, the system also computes a prediction score, which is converted into a health percentage to provide a more intuitive understanding of machine performance.
To extend beyond simple anomaly detection, the system incorporates a deep learning-based forecasting mechanism using sequence models. These models are trained to analyze temporal patterns in historical data and predict future values of machine parameters. This capability is particularly valuable in industrial settings, as it allows operators to anticipate failures before they occur and take preventive actions. By combining real-time monitoring with forward-looking predictions, the system moves from a reactive approach to a proactive maintenance strategy.
The processed results are visualized through an interactive dashboard built using Streamlit, which serves as the primary user interface of the system. The dashboard is designed to provide both high-level and detailed insights in an intuitive and visually accessible manner. At a global level, users can view an overview of all machines, including the total count, the number of machines operating normally, and those flagged with anomalies. Each machine is displayed with a health indicator and status label, allowing quick identification of potential issues.
For deeper analysis, the dashboard includes a dedicated machine view where users can inspect individual machine performance in detail. This includes real-time sensor readings, dynamically updating trend graphs, and a historical view of past data. Additionally, the system displays forecasted values generated by the deep learning model, enabling users to understand how machine behavior is expected to evolve. An alerts section further enhances the monitoring capability by logging anomaly events along with timestamps and key parameter values, creating a simple yet effective tracking system for critical incidents.
An important aspect of this project is its integration with ThingsBoard, an IoT platform used for visualizing telemetry data in real-world industrial applications. Instead of limiting the project to static or simulated dashboards, the system pushes data to ThingsBoard and provides access to a live public dashboard. This adds a layer of realism to the project, demonstrating how such systems are deployed in production environments where stakeholders rely on external dashboards for monitoring and decision-making.
From a technical standpoint, the project demonstrates the integration of multiple technologies into a cohesive architecture. The backend is built using Flask for lightweight API handling, while data processing and manipulation are handled using Pandas and NumPy. Machine learning models are implemented using Scikit-learn for anomaly detection, and TensorFlow/Keras is used for building and loading deep learning models for forecasting. The frontend is developed using Streamlit, which enables rapid creation of interactive data applications without the need for complex frontend frameworks.
The project is structured in a modular manner, with clear separation between data simulation, backend logic, model handling, and frontend visualization. This modularity ensures that the system is scalable and maintainable. For instance, additional sensors can be introduced, new machine types can be simulated, or more advanced models can be integrated without requiring a complete redesign of the system. Such flexibility is essential for real-world industrial applications where systems evolve over time.
To run the project, the repository can be cloned and set up in a local environment by installing the required dependencies listed in the requirements file. The backend API is started first, which begins processing simulated data and serving predictions. The Streamlit dashboard is then launched to provide an interactive interface for users. The dashboard continuously communicates with the backend to fetch updated data, ensuring a near real-time experience. Additionally, users can access the integrated ThingsBoard dashboard through a provided public link to observe telemetry data in a production-style environment.
Overall, the Smart Mining Digital Twin System represents a complete implementation of a modern intelligent monitoring solution. It brings together concepts from machine learning, deep learning, data engineering, and frontend development into a single unified application. The project highlights the practical application of AI in industrial IoT scenarios and demonstrates how digital twin technology can be leveraged to improve operational efficiency, reduce downtime, and enable smarter decision-making.
This project also reflects strong problem-solving and system design capabilities, particularly in handling real-time data flows, integrating multiple components, and overcoming environment and deployment challenges. It serves as a solid example of how theoretical knowledge in data science and machine learning can be translated into a functional, real-world application.

## Features
- Real-time sensor data simulation (temperature, vibration, load, pressure)
- LSTM-based anomaly detection for machine health monitoring
- Multi-machine digital twin simulation
- Predictive forecasting using deep learning models
- Interactive dashboard built with Streamlit
- Real-time alerts and health score visualization
- Integration with ThingsBoard for live IoT dashboard

## Architecture
- Backend: Flask API for real-time data processing and prediction
- AI Model: LSTM (TensorFlow/Keras) for anomaly detection and forecasting
- Frontend: Streamlit dashboard for visualization and monitoring
- Data Layer: Simulated real-time sensor data generation
- Integration: ThingsBoard for external IoT dashboard visualization

## Machines Simulated
- Excavator
- Drill
- Truck

## System Workflow
- Sensor data is simulated in real time
- Data is sent to the Flask backend API
- ML/DL models process data for anomaly detection
- Health score and status are generated
- Results are displayed on Streamlit dashboard
- Telemetry is pushed to ThingsBoard for live monitoring

 ##How to Run the Project

1. Clone the repository:
   git clone https://https://github.com/eshaparuchuri/smart-mining-system

2. Navigate into the project folder:
   cd smart-mining-digital-twin

3. Create a virtual environment:
   python -m venv venv

4. Activate the environment:
   .\venv\Scripts\Activate.ps1

5. Install dependencies:
   pip install -r requirements.txt

6. Start the backend API:
   python -m backend.api

7. Launch the Streamlit dashboard:
   python -m streamlit run dashboard/app.py
